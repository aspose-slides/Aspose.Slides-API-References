### AccentProcess {#AccentProcess}
```
public static final int AccentProcess
```

Verwenden, um einen Fortschritt, einen Zeitstrahl oder sequenzielle Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen. Eignet sich gut, um sowohl Level-1- als auch Level-2-Text zu illustrieren.

### AccentedPicture {#AccentedPicture}
```
public static final int AccentedPicture
```

Verwenden, um eine zentrale fotografische Idee mit zugehörigen Ideen an der Seite darzustellen. Der obere Level-1-Text erscheint über dem zentralen Bild. Der zugehörige Text für andere Level-1-Formen erscheint neben den kleinen runden Bildern. Dieses Layout funktioniert auch gut ohne Text.

### AlternatingFlow {#AlternatingFlow}
```
public static final int AlternatingFlow
```

Verwenden, um Gruppen von Informationen oder sequenzielle Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen. Betont die Interaktion oder Beziehungen zwischen den Informationsgruppen.

### AlternatingHexagons {#AlternatingHexagons}
```
public static final int AlternatingHexagons
```

Verwenden, um eine Reihe miteinander verbundener Ideen darzustellen. Level-1-Text erscheint in den Hexagonen. Level-2-Text erscheint außerhalb der Formen.

### AlternatingPictureBlocks {#AlternatingPictureBlocks}
```
public static final int AlternatingPictureBlocks
```

Verwenden, um eine Reihe von Bildern von oben nach unten darzustellen. Der Text erscheint abwechselnd rechts oder links vom Bild.

### AlternatingPictureCircles {#AlternatingPictureCircles}
```
public static final int AlternatingPictureCircles
```

Verwenden, um einen Satz von Bildern mit Text darzustellen. Der zugehörige Text erscheint in den zentralen Kreisen, wobei die Bilder abwechselnd von links nach rechts angeordnet sind.

### ArrowRibbon {#ArrowRibbon}
```
public static final int ArrowRibbon
```

Verwenden, um verwandte oder kontrastierende Konzepte mit einer Verbindung darzustellen, z. B. gegensätzliche Kräfte. Die ersten beiden Zeilen Level-1-Text werden für den Text in den Pfeilen verwendet. Unbenutzter Text wird nicht angezeigt, bleibt aber verfügbar, wenn das Layout gewechselt wird.

### AscendingPictureAccentProcess {#AscendingPictureAccentProcess}
```
public static final int AscendingPictureAccentProcess
```

Verwenden, um eine aufsteigende Reihe von Bildern mit beschreibendem Text darzustellen. Am besten geeignet für eine kleine Textmenge.

### Balance {#Balance}
```
public static final int Balance
```

Verwenden, um die Beziehung zwischen zwei Ideen zu vergleichen oder darzustellen. Jede der ersten beiden Zeilen Level-1-Text entspricht dem Text oben an einer Seite des Mittelpunktes. Betont Level-2-Text, der auf vier Formen pro Seite des Mittelpunktes beschränkt ist. Das Gleichgewicht kippt zur Seite mit den meisten Formen, die Level-2-Text enthalten. Unbenutzter Text wird nicht angezeigt, bleibt aber verfügbar, wenn das Layout gewechselt wird.

### BasicBendingProcess {#BasicBendingProcess}

public static final int BasicBendingProcess

Verwenden, um einen Fortschritt oder sequenzielle Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen. Nutzt sowohl horizontalen als auch vertikalen Anzeigeraum für Formen maximal aus.

### BasicBlockList {#BasicBlockList}

public static final int BasicBlockList

### BasicPie {#BasicPie}
```
public static final int BasicPie
```

Der obere Level-1-Text erscheint außerhalb des restlichen Kuchens zur Betonung. Unbenutzter Text wird nicht angezeigt, bleibt aber verfügbar, wenn das Layout gewechselt wird.

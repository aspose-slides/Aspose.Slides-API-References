## ILinkEmbedController.GetObjectStoringLocation متد

مشخص می‌کند که شی در کجا باید ذخیره شود. این متد یک بار برای هر شناسهٔ شی فراخوانی می‌شود. تضمین نمی‌شود که دو شی با داده‌های یکسان، semanticName و contentType ولی شناسه‌های متفاوت وجود نداشته باشند.

```csharp
public LinkEmbedDecision GetObjectStoringLocation(int id, byte[] entityData, string semanticName, 
    string contentType, string recomendedExtension)
```

| پارامتر | نوع | توضیح |
| --- | --- | --- |
| id | Int32 | شناسهٔ شی. این شناسه به‌صورت یکتا در سراسر عملیات ذخیره‌سازی است. |
| entityData | Byte[] | دادهٔ باینری شی. این پارامتر می‌تواند null باشد، اگر دادهٔ باینری شی هنوز تولید نشده باشد. |
| semanticName | String | متنی کوتاه که معنای شی را توصیف می‌کند. کنترلر ممکن است از این به‌عنوان بخشی از نام شی خارجی استفاده کند، اما اطمینان از یکتا بودن نام‌ها و داشتن فقط کاراکترهای مجاز بر عهدهٔ dispatcher است. |
| contentType | String | نوع MIME شی. |
| recomendedExtension | String | پسوند نام فایل که برای این نوع MIME توصیه می‌شود. |

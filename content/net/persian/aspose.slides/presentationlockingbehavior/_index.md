## PresentationLockingBehavior شمارش

رفتاری را که در هنگام بارگذاری و کار با یک نمونه از [`IPresentation`](../ipresentation) برای منبع [`IPresentation`](../ipresentation) (فایل یا Stream) اتخاذ می‌شود، نشان می‌دهد.

```csharp
public enum PresentationLockingBehavior
```

### مقادیر

| نام | مقدار | توضیح |
| --- | --- | --- |
| LoadAndRelease | `0` | منبع فقط برای مدت زمان اجرای سازنده [`IPresentation`](../ipresentation) قفل می‌شود. اگر [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed) برابر false تنظیم شود، تمام BLOBها در حافظه بارگذاری می‌شوند. در غیر این صورت، روش‌های دیگری مانند فایل‌های موقت ممکن است استفاده شوند. این رفتار نسبت به KeepLocked کندتر است و اگر امکان انتقال مالکیت منبع به [`IPresentation`](../ipresentation) وجود داشته باشد، استفاده از KeepLocked توصیه می‌شود. |
| KeepLocked | `1` | منبع برای تمام طول عمر نمونهٔ [`IPresentation`](../ipresentation) قفل می‌شود تا زمانی که آن رها شود. برای استفاده از این رفتار باید [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed) برابر true تنظیم شود، در غیر این صورت استثنا پرتاب خواهد شد. این رفتار توصیه می‌شود، سریع‌تر است و نسبت به LoadAndRelease حافظه کمتری مصرف می‌کند. |

### نکات

منبع پارامتری است که به سازندهٔ [`IPresentation`](../ipresentation) ارسال می‌شود. در مثال زیر، منبع فایل "pres.pptx" است:

```csharp
LoadOptions loadOptions = new LoadOptions { 
  BlobManagementOptions = { PresentationLockingBehavior = PresentationLockingBehavior.KeepLocked } };
using (IPresentation pres = new Presentation("pres.pptx", loadOptions)) { }
```

برای این مثال، منبع (فایل "pres.pptx") برای طول عمر نمونهٔ [`IPresentation`](../ipresentation) قفل می‌شود، یعنی توسط فرآیند دیگر قابل تغییر یا حذف نیست.

## شمارش PresentationLockingBehavior

رفتاری را که در مورد پردازش منبع [`IPresentation`](../ipresentation) (فایل یا Stream) هنگام بارگذاری و کار با یک نمونه از [`IPresentation`](../ipresentation) توصیف می‌کند.

```csharp
public enum PresentationLockingBehavior
```

### مقادیر

| نام | مقدار | توضیح |
| --- | --- | --- |
| LoadAndRelease | `0` | منبع فقط برای مدت زمان اجرای سازنده [`IPresentation`](../ipresentation) قفل خواهد شد. اگر [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed) روی false تنظیم شود، تمام BLOBها در حافظه بارگذاری می‌شوند. در غیر این صورت، ممکن است از روش‌های دیگر مانند فایل‌های موقت استفاده شود. این رفتار نسبت به KeepLocked کندتر است و اگر امکان انتقال مالکیت منبع به [`IPresentation`](../ipresentation) وجود داشته باشد، استفاده از KeepLocked توصیه می‌شود. |
| KeepLocked | `1` | منبع برای تمام طول عمر نمونهٔ [`IPresentation`](../ipresentation) قفل خواهد شد، تا زمانی که از بین برود. [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed) باید برای استفاده از این رفتار روی true تنظیم شود، در غیر این صورت استثنا پرتاب می‌شود. این رفتار توصیه می‌شود، سریع‌تر بوده و نسبت به LoadAndRelease مصرف حافظه کمتری دارد. |

### ملاحظات

منبع پارامتری است که به سازندهٔ [`IPresentation`](../ipresentation) پاس داده می‌شود. در مثال زیر، منبع فایل "pres.pptx" است:

```csharp
LoadOptions loadOptions = new LoadOptions { 
  BlobManagementOptions = { PresentationLockingBehavior = PresentationLockingBehavior.KeepLocked } };
using (IPresentation pres = new Presentation("pres.pptx", loadOptions)) { }
```

برای این مثال، منبع (فایل "pres.pptx") برای تمام طول عمر نمونهٔ [`IPresentation`](../ipresentation) قفل خواهد شد، یعنی توسط فرآیند دیگر قابل تغییر یا حذف نیست.

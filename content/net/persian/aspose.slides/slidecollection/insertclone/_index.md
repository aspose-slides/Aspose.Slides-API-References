### مقدار بازگشتی

### توضیحات

هنگامی که اسلایدی را بین ارائه‌های مختلف کلون می‌کنید، مستر اسلاید نیز می‌تواند کلون شود. ثبت داخلی برای ردیابی مسترهای کلون شده به صورت خودکار استفاده می‌شود تا از ایجاد چندین کلون از همان مستر اسلاید جلوگیری کند. کلون کردن دستی مستر اسلایدها نه جلوگیری می‌شود و نه ثبت می‌شود. اگر به کنترل بیشتری در فرآیند کلون کردن نیاز دارید، از [`InsertClone`](../insertclone) یا [`InsertClone`](../insertclone) برای کلون کردن اسلایدها و [`AddClone`](../../imasterslidecollection/addclone) برای کلون کردن مسترها استفاده کنید.

### مثال‌ها

مثال زیر نشان می‌دهد چگونه اسلاید را در موقعیت دیگری در داخل Presentation کلون کنید.

```csharp
[C#]
// نمونه‌سازی کلاس Presentation که نمایانگر یک فایل ارائه است
using (Presentation pres = new Presentation("CloneWithInSamePresentation.pptx"))
{
    // اسلاید مورد نظر را به انتهای مجموعه اسلایدها در همان ارائه کلون می‌کند
    ISlideCollection slds = pres.Slides;
    // اسلاید مورد نظر را به ایندکس مشخص شده در همان ارائه کلون می‌کند
    slds.InsertClone(2, pres.Slides[1]);
    // ارائهٔ تغییر یافته را به دیسک ذخیره می‌کند
    pres.Save("Aspose_CloneWithInSamePresentation_out.pptx", SaveFormat.Pptx);
}
```

مثال زیر نشان می‌دهد چگونه اسلاید را در موقعیت دیگری در داخل Presentation کلون کنید.

```csharp
[C#]
// نمونه‌سازی کلاس Presentation برای بارگذاری فایل ارائه منبع
using (Presentation srcPres = new Presentation("CloneAtEndOfAnother.pptx"))
{
    // نمونه‌سازی کلاس Presentation برای فایل PPTX مقصد (جایی که اسلاید باید کلون شود)
    using (Presentation destPres = new Presentation())
    {
        ISlideCollection slds = destPres.Slides;
        slds.InsertClone(2, srcPres.Slides[0]);
        // ارائهٔ مقصد را در دیسک ذخیره می‌کند
        destPres.Save("Aspose2_out.pptx", SaveFormat.Pptx);
    }
}
```

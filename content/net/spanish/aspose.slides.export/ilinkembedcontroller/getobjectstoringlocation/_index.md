## ILinkEmbedController.GetObjectStoringLocation method

Determina dónde se debe almacenar el objeto. Este método se llama una vez para cada id. de objeto. No se garantiza que no habrá dos objetos con los mismos datos, semanticName y contentType pero con diferente id.

```csharp
public LinkEmbedDecision GetObjectStoringLocation(int id, byte[] entityData, string semanticName, 
    string contentType, string recomendedExtension)
```

| Parámetro | Escribe | Descripción |
| --- | --- | --- |
| id | Int32 | identificación del objeto Esta identificación es única en toda la operación. |
| entityData | Byte[] | Objeto de datos binarios. Este parámetro puede ser nulo, si los datos binarios del objeto aún no se generan. |
| semanticName | String | Algún texto breve que describa el significado del objeto. El controlador puede usar esto como parte del nombre del objeto externo, pero depende del despachador asegurarse de que los nombres sean únicos y contengan solo los caracteres permitidos. |
| contentType | String | Tipo de objeto MIME. |
| recomendedExtension | String | Extensión de nombre de archivo, recomendada para este tipo MIME. |

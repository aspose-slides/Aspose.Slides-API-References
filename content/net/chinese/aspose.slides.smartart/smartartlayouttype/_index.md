## SmartArtLayoutType 枚举

表示 SmartArt 图表的布局类型。

```csharp
public enum SmartArtLayoutType
```

### 值

| 名称 | 值 | 描述 |
| --- | --- | --- |
| AccentProcess | `0` | 用于展示任务、流程或工作流中的进程、时间线或顺序步骤。适用于同时插入一级和二级文本。 |
| AccentedPicture | `1` | 用于展示一个中央的、摄影而成的想法，以及旁边相关的想法。顶部的一级文本出现在中央图片上方。其他一级形状的相应文本出现在小圆形图片旁边。此布局在没有文本时也很好。 |

| BasicMatrix | `13` | 用于展示组件与整体的关系。前四行的一级文本出现在四个象限中。未使用的文本不会出现，但如果切换布局，仍然可用。 |

| BasicTarget | `18` | 用于展示包含关系、渐变或层次关系。前五行的一级文本与一个圆形相关联。未使用的文本不会出现，但如果切换布局，仍然可用。 |

| BendingPictureCaption | `23` | 用于展示顺序系列的图片。覆盖底角的框可以包含少量文本。 |
| BendingPictureCaptionList | `24` | 用于展示一系列的图片。标题和描述出现在每张图片下方的气泡形状中。 |

| BlockCycle | `26` | 用于表示在循环流中持续的阶段、任务或事件序列。强调阶段或步骤，而非连接的箭头或流。 |
| BubblePictureList | `27` | 用于展示一系列图片。最多可包含八个一级图片。未使用的文本和图片不会出现，但如果切换布局，仍然可用。适合少量文本。 |
| CaptionedPictures | `28` | 用于展示带有多个级别文本的图片。最适合少量一级文本和中等数量的二级文本。 |
| ChevronList | `29` | 用于展示通过多个过程的进程，这些过程构成了总体工作流。也适合用于展示对比过程。一级文本对应于左侧的第一个箭头形状，而二级文本对应于每个包含一级文本形状的水平子步骤。 |
| CircleAccentTimeline | `30` | 用于展示一系列事件或时间线信息。一级文本出现在较大的圆形旁。二级文本出现在较小的圆形旁。 |

| ContinuousBlockProcess | `38` | 用于展示任务、过程或工作流中的进程或顺序步骤。最适合最小的一级和二级文本。 |
| ContinuousCycle | `39` | 用于表示在循环流中持续的阶段、任务或事件序列。强调所有组件之间的连接。最适合仅使用一级文本。 |

| ConvergingArrows | `41` | 用于展示汇聚到一个中心点的想法或概念。最适合仅使用一级文本。 |

| GroupedList | `55` | 用于展示信息的组和子组，或在任务、过程或工作流中的步骤和子步骤。一级文本对应于顶部水平形状，二级文本对应于每个相关顶部形状下的垂直子步骤。适合强调子组或子步骤、层次信息或多组信息。 |

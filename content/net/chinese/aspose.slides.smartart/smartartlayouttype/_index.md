## SmartArtLayoutType 枚举

表示 SmartArt 图表的布局类型。

```csharp
public enum SmartArtLayoutType
```

### 值

| 名称 | 值 | 描述 |
| --- | --- | --- |
| AccentProcess | `0` | 用于显示进程、时间线或任务、过程或工作流中的顺序步骤。适合用于说明一级和二级文本。 |
| AccentedPicture | `1` | 用于展示一个中心的摄影想法以及相关的旁边想法。顶部的一级文本出现在中央图片上方。相应的其他一级形状的文本出现在小圆形图片旁边。该布局在没有文本的情况下也很好用。 |
| AlternatingFlow | `2` | 用于显示信息组或任务、过程或工作流中的顺序步骤。强调信息组之间的互动或关系。 |

| AlternatingPictureBlocks | `4` | 用于从上到下展示一系列图片。文本交替出现在图片的右侧或左侧。 |
| AlternatingPictureCircles | `5` | 用于展示一组带有文本的图片。相应的文本出现在中央圆圈中，图片则从左到右交替排列。 |

| BasicBendingProcess | `9` | 用于展示任务、过程或工作流中的进程或顺序步骤。最大化形状的水平和垂直显示空间。 |

| BasicCycle | `12` | 用于表示阶段、任务或事件的持续序列，流动呈循环。强调阶段或步骤而非连接箭头或流动。仅使用一级文本效果最佳。 |
| BasicMatrix | `13` | 用于展示组件与整体的关系，以象限形式展示。前四行的一级文本出现在象限中。未使用的文本不显示，但如果切换布局仍然可用。 |
| BasicPie | `14` | 用于展示各个部分形成整体的情况。前七行的一级文本对应于均匀分布的楔形或饼形。顶部的一级文本形状位于剩余饼形的外侧，以示强调。未使用的文本不显示，但如果切换布局仍然可用。 |

| BendingPictureAccentList | `21` | 用于展示非顺序或分组的信息块。小圆形形状设计用于包含图片。适合同时说明一级和二级文本。最大化形状的水平和垂直显示空间。 |

| BlockCycle | `26` | 用于表示阶段、任务或事件的持续序列，流动呈循环。强调阶段或步骤，而非连接箭头或流动。 |

| CircleAccentTimeline | `30` | 用于展示一系列事件或时间线信息。一级文本出现在较大的圆形旁边。二级文本出现在较小的圆形旁边。 |

| ContinuousBlockProcess | `38` | 用于展示任务、过程或工作流中的进程或顺序步骤。效果最好的是最少的一级和二级文本。 |
| ContinuousCycle | `39` | 用于表示阶段、任务或事件的持续序列，流动呈循环。强调各组件之间的连接。仅使用一级文本效果最佳。 |

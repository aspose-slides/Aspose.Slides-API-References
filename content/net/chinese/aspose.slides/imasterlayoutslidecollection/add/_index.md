## IMasterLayoutSlideCollection.Add 方法

将新的布局幻灯片添加到集合的末尾。

```csharp
public ILayoutSlide Add(SlideLayoutType layoutType, string layoutName)
```

| 参数 | 类型 | 描述 |
| --- | --- | --- |
| layoutType | SlideLayoutType | 新布局的布局类型。支持的布局类型：标题、仅标题、空白、标题和对象、垂直文本、垂直标题和文本、两个对象、部分标题、两个文本和两个对象、标题对象和说明、图片和说明、自定义。当前不支持的其他布局类型：文本、双列文本、表格、文本和图表、图表和文本、图示、图表、文本和剪贴画、剪贴画和文本、文本和对象、对象和文本、对象、文本和媒体、媒体和文本、覆盖文本的对象、覆盖对象的文本、文本和两个对象、两个对象和文本、两个对象覆盖文本、四个对象、剪贴画和垂直文本、垂直标题和文本叠加在图表上、对象和两个对象、两个对象和对象。 |
| layoutName | 字符串 | 新布局的名称。如果传递的名称已被使用，将抛出ArgumentException。如果传递了null参数，则根据传递的布局类型自动生成名称（例如“标题幻灯片”或“1_标题幻灯片”、“2_..”等）。 |

### 返回值

添加的幻灯片。

### 异常

| 异常 | 条件 |
| --- | --- |
| NotImplementedException | 如果传递了不支持的参数 *layoutType* 的值，则抛出。当前不支持的布局类型：文本、双列文本、表格、文本和图表、图表和文本、图示、图表、文本和剪贴画、剪贴画和文本、文本和对象、对象和文本、对象、文本和媒体、媒体和文本、覆盖文本的对象、覆盖对象的文本、文本和两个对象、两个对象和文本、两个对象覆盖文本、四个对象、剪贴画和垂直文本、垂直标题和文本叠加在图表上、对象和两个对象、两个对象和对象。 |
| ArgumentException | 如果布局名称值 *layoutName* 已在该布局集合中使用，则抛出。 |

### 备注

1) 对于值 SlideLayoutType.Custom 的 *layoutType*，添加的布局不包含占位符和形状。 2) 该方法的类似方法是通过 [`LayoutSlides`](../../ipresentation/layoutslides) 属性访问的[`Add`](../../igloballayoutslidecollection/add) 方法。

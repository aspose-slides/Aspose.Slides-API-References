## LoadingStreamBehavior 列舉

傳遞給方法的 Stream 被視為二進位大型物件 (BLOB)（請參閱 [`IBlobManagementOptions`](../iblobmanagementoptions) 說明）。此列舉的值用來識別當 Stream 被傳遞給方法時該如何處理。根據需求，可以做出不同的決策以提供最高效的行為。

```csharp
public enum LoadingStreamBehavior
```

### 值

| Name | Value | Description |
| --- | --- | --- |
| ReadStreamAndRelease | `0` | 該串流將被讀取至結束，然後釋放 ─ 也就是保證未來此串流不會再被 [`IPresentation`](../ipresentation) 實例使用。它可以由客戶端程式碼關閉或以其他任何方式使用。 |
| KeepLocked | `1` | 該串流將在 [`IPresentation`](../ipresentation) 物件內被鎖定，也就是串流的所有權將被轉移。[`IPresentation`](../ipresentation) 物件將負責在自身被處置時正確釋放該串流。當需要序列化大型 BLOB 檔案（例如大型視訊或音訊──請參閱 [`IBlobManagementOptions`](../iblobmanagementoptions) 說明）且想避免將此檔案載入記憶體或產生其他效能問題時，此行為極為有用。您只需要為該檔案開啟 FileStream，並將其傳遞給方法，同時選擇 KeepLocked LoadingStreamBehavior 即可。 |

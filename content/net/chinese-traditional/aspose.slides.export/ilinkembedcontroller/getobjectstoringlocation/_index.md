## ILinkEmbedController.GetObjectStoringLocation 方法

決定物件應該儲存的位置。此方法會為每個物件 id 呼叫一次。無法保證不會有兩個具有相同 data、semanticName 和 contentType 但 id 不同的物件。

```csharp
public LinkEmbedDecision GetObjectStoringLocation(int id, byte[] entityData, string semanticName, 
    string contentType, string recomendedExtension)
```

| 參數 | 類型 | 說明 |
| --- | --- | --- |
| id | Int32 | 物件 id。此 id 在整個儲存操作中是唯一的。 |
| entityData | Byte[] | 物件二進位資料。若物件二進位資料尚未產生，此參數可為 null。 |
| semanticName | String | 一些簡短的文字，用於描述物件的意義。Controller 可能會將其作為外部物件名稱的一部分，但由 dispatcher 負責確保名稱唯一且僅包含允許的字元。 |
| contentType | String | 物件的 MIME 類型。 |
| recomendedExtension | String | 此 MIME 類型推薦的檔案副檔名。 |

### 返回值

決策

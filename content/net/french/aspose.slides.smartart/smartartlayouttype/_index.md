## Énumération SmartArtLayoutType

Représente le type de mise en page d'un diagramme SmartArt.

```csharp
public enum SmartArtLayoutType
```

### Valeurs

| Nom | Valeur | Description |
| --- | --- | --- |
| AccentProcess | `0` | Utilisé pour montrer une progression, une chronologie ou des étapes séquentielles dans une tâche, un processus ou un flux de travail. Fonctionne bien pour illustrer à la fois le texte de Niveau 1 et de Niveau 2. |
| AccentedPicture | `1` | Utilisé pour montrer une idée centrale, photographique, avec des idées connexes sur le côté. Le texte de Niveau 1 apparaît au-dessus de l'image centrale. Le texte correspondant aux autres formes de Niveau 1 apparaît à côté des petites images circulaires. Cette mise en page fonctionne également bien sans texte. |
| AlternatingFlow | `2` | Utilisé pour montrer des groupes d'informations ou des étapes séquentielles dans une tâche, un processus ou un flux de travail. Met l'accent sur l'interaction ou les relations entre les groupes d'informations. |
| AlternatingHexagons | `3` | Utilisé pour représenter une série d'idées interconnectées. Le texte de Niveau 1 apparaît à l'intérieur des hexagones. Le texte de Niveau 2 apparaît à l'extérieur des formes. |
| AlternatingPictureBlocks | `4` | Utilisé pour montrer une série de photos de haut en bas. Le texte apparaît alternativement à droite ou à gauche de l'image. |
| AlternatingPictureCircles | `5` | Utilisé pour montrer un ensemble de photos avec du texte. Le texte correspondant apparaît dans les cercles centraux avec les images alternant de gauche à droite. |
| ArrowRibbon | `6` | Utilisé pour montrer des concepts liés ou contrastés avec une certaine connexion, comme des forces opposées. | AscendingPictureAccentProcess | `7` | Utilisé pour montrer une série ascendante de photos avec du texte descriptif. Fonctionne mieux avec une petite quantité de texte. |

Le texte inutilisé n'apparaît pas, mais reste disponible si vous changez de mise en page. |
| BasicBendingProcess | `9` | Utilisé pour montrer une progression ou des étapes séquentielles dans une tâche, un processus ou un flux de travail. Maximiser l'espace d'affichage horizontal et vertical pour les formes. |

| BasicChevronProcess | `11` | Utilisé pour montrer une progression ; une chronologie ; des étapes séquentielles dans une tâche, un processus ou un flux de travail ; ou pour souligner le mouvement ou la direction. | BasicCycle | `12` | Utilisé pour représenter une séquence continue d'étapes, de tâches ou d'événements dans un flux circulaire. Met l'accent sur les étapes plutôt que sur les flèches ou le flux connectant. Fonctionne mieux avec uniquement du texte de Niveau 1. |
| BasicMatrix | `13` | Utilisé pour montrer la relation des composants à un tout dans des quadrants. Les quatre premières lignes de texte de Niveau 1 apparaissent dans les quadrants. Le texte inutilisé n'apparaît pas, mais reste disponible si vous changez de mise en page. |
| BasicPie | `14` | Utilisé pour montrer comment des parties individuelles forment un tout. | BasicProcess | `15` | Utilisé pour montrer une progression ou des étapes séquentielles dans une tâche, un processus ou un flux de travail. |
| BasicPyramid | `16` | Utilisé pour montrer des relations proportionnelles, interconnectées ou hiérarchiques avec le plus grand composant en bas et un rétrécissement vers le haut. | BasicRadial | `17` | Utilisé pour montrer la relation à une idée centrale dans un cycle. Le texte inutilisé n'apparaît pas, mais reste disponible si vous changez de mise en page. |
| BasicTarget | `18` | Utilisé pour montrer les relations de containment, de gradation ou hiérarchiques. Les cinq premières lignes de texte de Niveau 1 sont associées à un cercle. Le texte inutilisé n'apparaît pas, mais reste disponible si vous changez de mise en page. |
| BasicTimeline | `19` | Utilisé pour montrer des étapes séquentielles dans une tâche, un processus ou un flux de travail, ou pour montrer des informations chronologiques. Fonctionne bien avec à la fois du texte de Niveau 1 et de Niveau 2. |
| BasicVenn | `20` | Utilisé pour montrer des relations qui se chevauchent ou interconnectées. Les sept premières lignes de texte de Niveau 1 correspondent à un cercle. Le texte inutilisé n'apparaît pas, mais reste disponible si vous changez de mise en page. |
| BendingPictureAccentList | `21` | Utilisé pour montrer des blocs d'informations non séquentiels ou groupés. Les petites formes circulaires sont conçues pour contenir des images. | BendingPictureBlocks | `22` | Utilisé pour montrer une série de photos. La boîte couvrant le coin inférieur peut contenir de petites quantités de texte. |

| BendingPictureCaptionList | `24` | Utilisé pour montrer une série de photos. Le titre et la description apparaissent dans une forme d'appel sous chaque image. |
| BendingPictureSemiTransparentText | `25` | Utilisé pour montrer une série de photos. Une boîte semi-transparente couvre la partie inférieure de l'image et contient tous les niveaux de texte. |
| BlockCycle | `26` | Utilisé pour représenter une séquence continue d'étapes, de tâches ou d'événements dans un flux circulaire. Met l'accent sur les étapes plutôt que sur les flèches ou le flux connectant. |
| BubblePictureList | `27` | Utilisé pour montrer une série de photos. Peut contenir jusqu'à huit images de Niveau 1. Le texte et les images inutilisés n'apparaissent pas, mais restent disponibles si vous changez de mise en page. Fonctionne mieux avec de petites quantités de texte. |

| CircleAccentTimeline | `30` | Utilisé pour montrer une série d'événements ou des informations chronologiques. Le texte de Niveau 1 apparaît à côté de formes circulaires plus grandes. Le texte de Niveau 2 apparaît à côté de formes circulaires plus petites. |

| CirclePictureHierarchy | `32` | Utilisé pour montrer des informations hiérarchiques ou des relations de reporting dans une organisation. Les images apparaissent dans des cercles et le texte correspondant apparaît à côté des images. |
| CircleRelationship | `33` | Utilisé pour montrer la relation à ou depuis une idée centrale. Le texte de Niveau 2 est ajouté de manière non séquentielle et est limité à cinq éléments. Il ne peut y avoir qu'un seul élément de Niveau 1. |
| CircularBendingProcess | `34` | Utilisé pour montrer une séquence longue ou non linéaire ou des étapes dans une tâche, un processus ou un flux de travail. Fonctionne mieux avec uniquement du texte de Niveau 1. Maximiser l'espace d'affichage horizontal et vertical pour les formes. |
| CircularPictureCallout | `35` | Utilisé pour montrer une idée centrale et des sous-idées ou objets connexes. Le texte de la première image couvre la partie inférieure de l'image. | ClosedChevronProcess | `36` | Utilisé pour montrer une progression, une chronologie, ou des étapes séquentielles dans une tâche, un processus ou un flux de travail, ou pour souligner le mouvement ou la direction. | ContinuousArrowProcess | `37` | Utilisé pour montrer une chronologie ou des étapes séquentielles dans une tâche, un processus ou un flux de travail. | ContinuousBlockProcess | `38` | Utilisé pour montrer une progression ou des étapes séquentielles dans une tâche, un processus ou un flux de travail. Fonctionne mieux avec un minimum de texte de Niveau 1 et de Niveau 2. |
| ContinuousCycle | `39` | Utilisé pour représenter une séquence continue d'étapes, de tâches ou d'événements dans un flux circulaire. Met l'accent sur la connexion entre tous les composants. Fonctionne mieux avec uniquement du texte de Niveau 1. |
| ContinuousPictureList | `40` | Utilisé pour montrer des groupes d'informations interconnectées. Les formes circulaires sont conçues pour contenir des images. |

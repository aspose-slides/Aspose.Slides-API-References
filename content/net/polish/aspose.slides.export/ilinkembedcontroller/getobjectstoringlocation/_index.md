## ILinkEmbedController.GetObjectStoringLocation metoda

Określa, gdzie obiekt powinien być przechowywany. Ta metoda jest wywoływana raz dla każdego identyfikatora obiektu. Nie ma gwarancji, że nie będzie dwóch obiektów o tych samych danych, semanticName i contentType, ale z różnym identyfikatorem.

```csharp
public LinkEmbedDecision GetObjectStoringLocation(int id, byte[] entityData, string semanticName, 
    string contentType, string recomendedExtension)
```

| Parametr | Typ | Opis |
| --- | --- | --- |
| id | Int32 | Identyfikator obiektu. Ten identyfikator jest unikalny w całej operacji zapisu. |
| entityData | Byte[] | Dane binarne obiektu. Ten parametr może być null, jeśli dane binarne obiektu nie zostały jeszcze wygenerowane. |
| semanticName | String | Krótki tekst opisujący znaczenie obiektu. Kontroler może używać tego jako części zewnętrznej nazwy obiektu, ale to dispatcher zapewnia, że nazwy będą unikalne i zawierać tylko dozwolone znaki. |
| contentType | String | Typ MIME obiektu. |
| recomendedExtension | String | Rozszerzenie nazwy pliku rekomendowane dla tego typu MIME. |

Decyzja

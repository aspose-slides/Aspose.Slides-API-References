## PresentationLockingBehavior enumeracja

Reprezentuje zachowanie dotyczące traktowania źródła [`IPresentation`](../ipresentation) (plik lub Stream) podczas ładowania i pracy z instancją [`IPresentation`](../ipresentation).

```csharp
public enum PresentationLockingBehavior
```

### Wartości

| Nazwa | Wartość | Opis |
| --- | --- | --- |
| LoadAndRelease | `0` | Źródło będzie zablokowane tylko na czas wykonywania konstruktora [`IPresentation`](../ipresentation). Jeśli [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed) jest ustawione na false, wszystkie BLOB-y zostaną załadowane do pamięci. W przeciwnym razie mogą zostać użyte inne środki, takie jak pliki tymczasowe. To zachowanie jest wolniejsze niż KeepLocked i jeśli istnieje możliwość przekazania własności źródła do [`IPresentation`](../ipresentation), zaleca się użycie KeepLocked. |
| KeepLocked | `1` | Źródło będzie zablokowane przez cały okres życia instancji [`IPresentation`](../ipresentation), aż do jej zwolnienia. [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed) musi być ustawione na true, aby używać tego zachowania, w przeciwnym razie zostanie rzucony wyjątek. To zachowanie jest zalecane, jest szybsze i zużywa mniej pamięci niż LoadAndRelease. |

### Uwagi

Źródło jest parametrem przekazywanym do konstruktora [`IPresentation`](../ipresentation). W poniższym przykładzie źródłem jest plik „pres.pptx”:

```csharp
LoadOptions loadOptions = new LoadOptions { 
  BlobManagementOptions = { PresentationLockingBehavior = PresentationLockingBehavior.KeepLocked } };
using (IPresentation pres = new Presentation("pres.pptx", loadOptions)) { }
```

W tym przykładzie źródło (plik „pres.pptx”) będzie zablokowane na cały okres życia instancji [`IPresentation`](../ipresentation), tzn. nie może być zmieniane ani usuwane przez inny proces.

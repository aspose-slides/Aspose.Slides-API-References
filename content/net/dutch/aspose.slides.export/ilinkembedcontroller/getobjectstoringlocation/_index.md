## ILinkEmbedController.GetObjectStoringLocation methode

Bepaalt waar het object moet worden opgeslagen. Deze methode wordt één keer aangeroepen voor elke object-id. Het is niet gegarandeerd dat er niet twee objecten bestaan met dezelfde gegevens, semanticName en contentType maar met een verschillend id.

```csharp
public LinkEmbedDecision GetObjectStoringLocation(int id, byte[] entityData, string semanticName, 
    string contentType, string recomendedExtension)
```

| Parameter | Type | Beschrijving |
| --- | --- | --- |
| id | Int32 | Object-id. Deze id is uniek voor de gehele bewerking. |
| entityData | Byte[] | Binaire objectgegevens. Deze parameter kan null zijn als de binaire objectgegevens nog niet zijn gegenereerd. |
| semanticName | String | Korte tekst die de betekenis van het object beschrijft. De controller kan dit gebruiken als onderdeel van de externe objectnaam, maar het is aan de dispatcher om te zorgen dat namen uniek zijn en alleen toegestane tekens bevatten. |
| contentType | String | MIME-type van het object. |
| recomendedExtension | String | Bestandsextensie die wordt aanbevolen voor dit MIME-type. |

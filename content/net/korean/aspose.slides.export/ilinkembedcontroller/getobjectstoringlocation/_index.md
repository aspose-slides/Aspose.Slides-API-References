## ILinkEmbedController.GetObjectStoringLocation 메서드

개체를 저장해야 하는 위치를 결정합니다. 이 메서드는 각 개체 ID마다 한 번 호출됩니다. 동일한 데이터, semanticName 및 contentType을 가지면서 다른 ID를 가진 두 개체가 존재할 가능성이 있음을 보장하지 않습니다.

```csharp
public LinkEmbedDecision GetObjectStoringLocation(int id, byte[] entityData, string semanticName, 
    string contentType, string recomendedExtension)
```

| 매개변수 | 형식 | 설명 |
| --- | --- | --- |
| id | Int32 | 개체 ID. 이 ID는 저장 작업 전체에서 고유합니다. |
| entityData | Byte[] | 개체 바이너리 데이터. 아직 개체 바이너리 데이터가 생성되지 않은 경우 이 매개변수는 null일 수 있습니다. |
| semanticName | String | 개체의 의미를 설명하는 짧은 텍스트. 컨트롤러는 이를 외부 개체 이름의 일부로 사용할 수 있지만, 이름이 고유하고 허용된 문자만 포함하도록 보장하는 것은 디스패처의 책임입니다. |
| contentType | String | 개체의 MIME 유형. |
| recomendedExtension | String | 해당 MIME 유형에 권장되는 파일 이름 확장자. |

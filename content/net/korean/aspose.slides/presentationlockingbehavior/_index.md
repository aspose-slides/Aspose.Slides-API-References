## PresentationLockingBehavior 열거형

[`IPresentation`](../ipresentation) 소스(파일 또는 Stream)를 로드하고 [`IPresentation`](../ipresentation) 인스턴스로 작업할 때 처리하는 동작을 나타냅니다.

```csharp
public enum PresentationLockingBehavior
```

### 값

| 이름 | 값 | 설명 |
| --- | --- | --- |
| LoadAndRelease | `0` | 소스는 [`IPresentation`](../ipresentation) 생성자 실행 시간 동안만 잠깁니다. [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed)가 false로 설정되면 모든 BLOB이 메모리로 로드됩니다. 그렇지 않은 경우 임시 파일과 같은 다른 방법이 사용될 수 있습니다. 이 동작은 KeepLocked보다 느리며, 소스의 소유권을 [`IPresentation`](../ipresentation)에 전달할 수 있는 경우 KeepLocked을 사용하는 것이 권장됩니다. |
| KeepLocked | `1` | 소스는 [`IPresentation`](../ipresentation) 인스턴스의 전체 수명 동안, 폐기될 때까지 잠깁니다. 이 동작을 사용하려면 [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed)를 true로 설정해야 하며, 그렇지 않으면 예외가 발생합니다. 이 동작은 권장되며, LoadAndRelease보다 빠르고 메모리 사용량이 적습니다. |

### 비고

[`IPresentation`](../ipresentation) 생성자에 전달되는 매개변수가 소스입니다. 아래 예제에서 소스는 "pres.pptx" 파일입니다:

```csharp
LoadOptions loadOptions = new LoadOptions { 
  BlobManagementOptions = { PresentationLockingBehavior = PresentationLockingBehavior.KeepLocked } };
using (IPresentation pres = new Presentation("pres.pptx", loadOptions)) { }
```

이 예제에서는 소스("pres.pptx" 파일)가 [`IPresentation`](../ipresentation) 인스턴스 수명 동안 잠기게 되며, 다른 프로세스가 변경하거나 삭제할 수 없습니다.

## LoadingStreamBehavior 열거형

메서드에 전달된 Stream은 Binary Large Object (BLOB) 로 간주됩니다 ([`IBlobManagementOptions`](../iblobmanagementoptions) 설명 참조). 이 열거형의 값은 메서드에 전달될 때 Stream이 어떻게 처리되어야 하는지 지정합니다. 요구 사항에 따라 가장 효율적인 동작을 제공하기 위해 다양한 결정을 내릴 수 있습니다.

```csharp
public enum LoadingStreamBehavior
```

### 값

| 이름 | 값 | 설명 |
| --- | --- | --- |
| ReadStreamAndRelease | `0` | 스트림은 끝까지 읽힌 후 해제됩니다—즉, 향후 [`IPresentation`](../ipresentation) 인스턴스가 이 스트림을 사용하지 않음이 보장됩니다. 클라이언트 코드에서 스트림을 닫거나 다른 방식으로 사용할 수 있습니다. |
| KeepLocked | `1` | 스트림은 [`IPresentation`](../ipresentation) 객체 내부에 잠기게 되며, 즉 스트림의 소유권이 이전됩니다. [`IPresentation`](../ipresentation) 객체는 이 객체가 자체적으로 해제될 때 스트림을 올바르게 해제할 책임이 있습니다. 이 동작은 대용량 BLOB 파일(예: 대형 비디오 또는 오디오 파일—[`IBlobManagementOptions`](../iblobmanagementoptions) 설명 참조)을 직렬화해야 하고 해당 파일을 메모리로 로드하거나 기타 성능 문제를 방지하고자 할 때 매우 유용합니다. 파일에 대한 FileStream을 열고 메서드에 전달하면서 KeepLocked LoadingStreamBehavior를 선택하면 됩니다. |

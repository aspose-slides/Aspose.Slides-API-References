## PresentationLockingBehavior एन्यूमेरेशन

लोड करते समय और [`IPresentation`](../ipresentation) की एक इंस्टेंस के साथ काम करते समय [`IPresentation`](../ipresentation) स्रोत (फ़ाइल या Stream) को संभालने के संबंध में व्यवहार का प्रतिनिधित्व करता है।

```csharp
public enum PresentationLockingBehavior
```

### मान

| Name | Value | Description |
| --- | --- | --- |
| LoadAndRelease | `0` | स्रोत केवल [`IPresentation`](../ipresentation) कंस्ट्रक्टर निष्पादन के समय के लिए ही लॉक रहेगा। [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed) को false पर सेट करने पर, सभी BLOB मेमोरी में लोड हो जाएंगे। अन्यथा, अस्थायी फ़ाइलों जैसी अन्य विधियों का उपयोग किया जा सकता है। यह व्यवहार KeepLocked से धीमा है, और यदि स्रोत के स्वामित्व को [`IPresentation`](../ipresentation) को पास करना संभव हो, तो KeepLocked का उपयोग करने की सिफारिश की जाती है। |
| KeepLocked | `1` | स्रोत को [`IPresentation`](../ipresentation) इंस्टेंस के पूरे जीवनकाल के लिए लॉक किया जाएगा, जब तक वह नष्ट नहीं हो जाता। [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed) को इस व्यवहार के उपयोग के लिए true पर सेट किया जाना चाहिए, अन्यथा अपवाद फेंका जाएगा। यह व्यवहार अनुशंसित है, यह LoadAndRelease से तेज़ है और कम मेमोरी का उपयोग करता है। |

### टिप्पणी

स्रोत वह पैरामीटर है जो [`IPresentation`](../ipresentation) कंस्ट्रक्टर को पास किया जाता है। नीचे दिए गए उदाहरण में, स्रोत "pres.pptx" फ़ाइल है:

```csharp
LoadOptions loadOptions = new LoadOptions { 
  BlobManagementOptions = { PresentationLockingBehavior = PresentationLockingBehavior.KeepLocked } };
using (IPresentation pres = new Presentation("pres.pptx", loadOptions)) { }
```

इस उदाहरण के लिए, स्रोत ("pres.pptx" फ़ाइल) [`IPresentation`](../ipresentation) इंस्टेंस जीवनकाल के लिए लॉक रहेगा, अर्थात् इसे अन्य प्रक्रिया द्वारा बदला या हटाया नहीं जा सकता।

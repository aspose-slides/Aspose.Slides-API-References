## ILinkEmbedController.GetObjectStoringLocation विधि

निर्धारित करता है कि ऑब्जेक्ट को कहाँ संग्रहीत किया जाना चाहिए। यह विधि प्रत्येक ऑब्जेक्ट id के लिए एक बार कॉल की जाती है। यह गारंटी नहीं है कि दो ऑब्जेक्ट एक ही डेटा, semanticName और contentType के साथ लेकिन अलग id के साथ नहीं हो सकते।

```csharp
public LinkEmbedDecision GetObjectStoringLocation(int id, byte[] entityData, string semanticName, 
    string contentType, string recomendedExtension)
```

| पैरामीटर | प्रकार | विवरण |
| --- | --- | --- |
| id | Int32 | ऑब्जेक्ट id। यह id सहेजने के कार्य-व्यापी अद्वितीय है। |
| entityData | Byte[] | ऑब्जेक्ट बाइनरी डेटा। यदि ऑब्जेक्ट बाइनरी डेटा अभी तक निर्मित नहीं हुआ है, तो यह पैरामीटर null हो सकता है। |
| semanticName | String | ऑब्जेक्ट के अर्थ का वर्णन करने वाला कुछ छोटा पाठ। Controller इसको बाहरी ऑब्जेक्ट नाम का भाग के रूप में उपयोग कर सकता है, लेकिन यह dispatcher पर निर्भर करता है कि नाम अद्वितीय हों और केवल अनुमत अक्षर शामिल हों। |
| contentType | String | ऑब्जेक्ट का MIME प्रकार। |
| recomendedExtension | String | फ़ाइल नाम एक्सटेंशन, जो इस MIME प्रकार के लिए अनुशंसित है। |

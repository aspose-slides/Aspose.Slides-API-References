## ILinkEmbedController.GetObjectStoringLocation เมธอด

กำหนดตำแหน่งที่วัตถุควรถูกจัดเก็บ วิธีการนี้จะถูกเรียกหนึ่งครั้งสำหรับแต่ละรหัสวัตถุ ไม่ได้รับประกันว่าจะไม่มีวัตถุสองรายการที่มีข้อมูล, semanticName และ contentType เดียวกันแต่มีรหัสต่างกัน

```csharp
public LinkEmbedDecision GetObjectStoringLocation(int id, byte[] entityData, string semanticName, 
    string contentType, string recomendedExtension)
```

| พารามิเตอร์ | ประเภท | คำอธิบาย |
| --- | --- | --- |
| id | Int32 | รหัสวัตถุ. รหัสนี้เป็นรหัสที่ไม่ซ้ำกันทั่วการดำเนินการการบันทึก. |
| entityData | Byte[] | ข้อมูลไบนารีของวัตถุ. พารามิเตอร์นี้อาจเป็น null หากข้อมูลไบนารีของวัตถุตายังไม่ได้สร้าง. |
| semanticName | String | ข้อความสั้น ๆ ที่อธิบายความหมายของวัตถุ. ตัวควบคุมอาจใช้สิ่งนี้เป็นส่วนหนึ่งของชื่อวัตถุภายนอก, แต่ขึ้นอยู่กับ dispatcher ที่จะตรวจสอบให้แน่ใจว่าชื่อจะไม่ซ้ำและมีเฉพาะอักขระที่อนุญาต. |
| contentType | String | ประเภท MIME ของวัตถุ. |
| recomendedExtension | String | ส่วนขยายของชื่อไฟล์ที่แนะนำสำหรับประเภท MIME นี้. |

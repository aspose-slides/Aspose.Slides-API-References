## SmartArtLayoutType Aufzählung

Stellt den Layout-Typ eines SmartArt-Diagramms dar.

```csharp
public enum SmartArtLayoutType
```

### Werte

| Name | Wert | Beschreibung |
| --- | --- | --- |
| AccentProcess | `0` | Wird verwendet, um einen Fortschritt, eine Zeitleiste oder aufeinanderfolgende Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen. Eignet sich gut zur Veranschaulichung von sowohl Level 1- als auch Level 2-Text. |
| AccentedPicture | `1` | Wird verwendet, um eine zentrale, fotografische Idee mit verwandten Ideen an der Seite darzustellen. Der obere Level 1-Text erscheint über dem zentralen Bild. Entsprechender Text für andere Level 1-Formen erscheint neben den kleinen kreisförmigen Bildern. Dieses Layout funktioniert auch gut ohne Text. |
| AlternatingFlow | `2` | Wird verwendet, um Gruppen von Informationen oder aufeinanderfolgende Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen. Betont die Interaktion oder Beziehungen zwischen den Gruppen von Informationen. |
| AlternatingHexagons | `3` | Wird verwendet, um eine Reihe von verbundenen Ideen darzustellen. Level 1-Text erscheint innerhalb der Hexagone. Level 2-Text erscheint außerhalb der Formen. |
| AlternatingPictureBlocks | `4` | Wird verwendet, um eine Reihe von Bildern von oben nach unten darzustellen. Text erscheint abwechselnd rechts oder links vom Bild. |
| AlternatingPictureCircles | `5` | Wird verwendet, um eine Reihe von Bildern mit Text darzustellen. Der entsprechende Text erscheint in den zentralen Kreisen, wobei die Bilder abwechselnd von links nach rechts angeordnet sind. |

| AscendingPictureAccentProcess | `7` | Wird verwendet, um eine aufsteigende Reihe von Bildern mit beschreibendem Text darzustellen. Funktioniert am besten mit einer kleinen Menge an Text. |
| Balance | `8` | Wird verwendet, um zwei Ideen zu vergleichen oder die Beziehung zwischen ihnen darzustellen. Ungenutzter Text erscheint nicht, bleibt jedoch verfügbar, wenn Sie die Layouts wechseln. |
| BasicBendingProcess | `9` | Wird verwendet, um einen Fortschritt oder aufeinanderfolgende Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen. Maximiert sowohl den horizontalen als auch den vertikalen Anzeigebereich für Formen. |
| BasicBlockList | `10` | Wird verwendet, um nicht sequenzielle oder gruppierte Informationsblöcke darzustellen. Maximiert sowohl den horizontalen als auch den vertikalen Anzeigebereich für Formen. |
| BasicChevronProcess | `11` | Wird verwendet, um einen Fortschritt darzustellen; eine Zeitleiste; aufeinanderfolgende Schritte in einer Aufgabe, einem Prozess oder einem Workflow; oder um Bewegung oder Richtung zu betonen. Level 1-Text erscheint innerhalb einer Pfeilform, während Level 2-Text unter den Pfeilformen erscheint. |
| BasicCycle | `12` | Wird verwendet, um eine fortlaufende Sequenz von Phasen, Aufgaben oder Ereignissen in einem zirkulären Fluss darzustellen. Betont die Phasen oder Schritte und nicht die verbindenden Pfeile oder den Fluss. Funktioniert am besten nur mit Level 1-Text. |
| BasicMatrix | `13` | Wird verwendet, um die Beziehung von Komponenten zum Ganzen in Quadranten darzustellen. Die ersten vier Zeilen des Level 1-Texts erscheinen in den Quadranten. Ungenutzter Text erscheint nicht, bleibt jedoch verfügbar, wenn Sie die Layouts wechseln. |
| BasicPie | `14` | Wird verwendet, um zu zeigen, wie einzelne Teile ein Ganzes bilden. Die ersten sieben Zeilen des Level 1-Texts entsprechen den gleichmäßig verteilten Wedge- oder Kuchengestalten. Die oberste Level 1-Textform erscheint außerhalb des Restes des Kuchens zur Betonung. Ungenutzter Text erscheint nicht, bleibt jedoch verfügbar, wenn Sie die Layouts wechseln. |
| BasicProcess | `15` | Wird verwendet, um einen Fortschritt oder aufeinanderfolgende Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen. |
| BasicPyramid | `16` | Wird verwendet, um proportionale, miteinander verbundene oder hierarchische Beziehungen darzustellen, wobei die größte Komponente unten ist und sich nach oben verjüngt. | BasicRadial | `17` | Wird verwendet, um die Beziehung zu einer zentralen Idee in einem Zyklus darzustellen. Ungenutzter Text erscheint nicht, bleibt jedoch verfügbar, wenn Sie die Layouts wechseln. |
| BasicTarget | `18` | Wird verwendet, um Einschluss, Abstufungen oder hierarchische Beziehungen darzustellen. Die ersten fünf Zeilen des Level 1-Texts sind mit einem Kreis verbunden. Ungenutzter Text erscheint nicht, bleibt jedoch verfügbar, wenn Sie die Layouts wechseln. |
| BasicTimeline | `19` | Wird verwendet, um aufeinanderfolgende Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen oder um Zeitlinieninformationen anzuzeigen. Funktioniert gut mit sowohl Level 1- als auch Level 2-Text. |
| BasicVenn | `20` | Wird verwendet, um sich überschneidende oder miteinander verbundene Beziehungen darzustellen. Die ersten sieben Zeilen des Level 1-Texts entsprechen einem Kreis. Ungenutzter Text erscheint nicht, bleibt jedoch verfügbar, wenn Sie die Layouts wechseln. |
| BendingPictureAccentList | `21` | Wird verwendet, um nicht sequenzielle oder gruppierte Informationsblöcke darzustellen. Maximiert sowohl den horizontalen als auch den vertikalen Anzeigebereich für Formen. |
| BendingPictureBlocks | `22` | Wird verwendet, um eine Serie von Bildern darzustellen. Das Feld, das die untere Ecke abdeckt, kann kleine Mengen an Text enthalten. |
| BendingPictureCaption | `23` | Wird verwendet, um eine sequenzielle Serie von Bildern darzustellen. Das Feld, das die untere Ecke abdeckt, kann kleine Mengen an Text enthalten. |
| BendingPictureCaptionList | `24` | Wird verwendet, um eine Serie von Bildern darzustellen. Der Titel und die Beschreibung erscheinen in einer Sprechblase unter jedem Bild. |
| BendingPictureSemiTransparentText | `25` | Wird verwendet, um eine Serie von Bildern darzustellen. Ein halbtransparentes Feld bedeckt den unteren Teil des Bildes und enthält alle Textstufen. |
| BlockCycle | `26` | Wird verwendet, um eine fortlaufende Sequenz von Phasen, Aufgaben oder Ereignissen in einem zirkulären Fluss darzustellen. Betont die Phasen oder Schritte und nicht die verbindenden Pfeile oder den Fluss. |
| BubblePictureList | `27` | Wird verwendet, um eine Serie von Bildern darzustellen. Kann bis zu acht Level 1-Bilder enthalten. Ungenutzter Text und Bilder erscheinen nicht, bleiben jedoch verfügbar, wenn Sie die Layouts wechseln. Funktioniert am besten mit kleinen Mengen an Text. |
| CaptionedPictures | `28` | Wird verwendet, um Bilder mit mehreren Ebenen von Text darzustellen. Funktioniert am besten mit einer kleinen Menge an Level 1-Text und einer mittleren Menge an Level 2-Text. |
| ChevronList | `29` | Wird verwendet, um einen Fortschritt durch mehrere Prozesse darzustellen, die ein Gesamtarbeitsablauf ausmachen. | CircleAccentTimeline | `30` | Wird verwendet, um eine Serie von Ereignissen oder Zeitlinieninformationen darzustellen. Level 1-Text erscheint neben größeren kreisförmigen Formen. Level 2-Text erscheint neben kleineren kreisförmigen Formen. |
| CircleArrowProcess | `31` | Wird verwendet, um sequenzielle Elemente mit unterstützendem Text für jedes Element darzustellen. Dieses Diagramm funktioniert am besten mit kleinen Mengen an Level 1-Text. |
| CirclePictureHierarchy | `32` | Wird verwendet, um hierarchische Informationen oder Berichtsbeziehungen in einer Organisation darzustellen. Bilder erscheinen in Kreisen und der entsprechende Text erscheint neben den Bildern. |
| CircleRelationship | `33` | Wird verwendet, um die Beziehung zu oder von einer zentralen Idee darzustellen. Level 2-Text wird nicht sequenziell hinzugefügt und ist auf fünf Elemente begrenzt. Es kann nur ein Level 1-Element geben. |
| CircularBendingProcess | `34` | Wird verwendet, um eine lange oder nicht lineare Sequenz oder Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen. | CircularPictureCallout | `35` | Wird verwendet, um eine zentrale Idee sowie Unterideen oder verwandte Elemente darzustellen. Der Text für das erste Bild überlagert den unteren Teil des Bildes. | ClosedChevronProcess | `36` | Wird verwendet, um einen Fortschritt, eine Zeitleiste oder aufeinanderfolgende Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen oder um Bewegung oder Richtung zu betonen. | ContinuousArrowProcess | `37` | Wird verwendet, um eine Zeitleiste oder aufeinanderfolgende Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen. | ContinuousBlockProcess | `38` | Wird verwendet, um einen Fortschritt oder aufeinanderfolgende Schritte in einer Aufgabe, einem Prozess oder einem Workflow darzustellen. Funktioniert am besten mit minimalem Level 1- und Level 2-Text. |
| ContinuousCycle | `39` | Wird verwendet, um eine fortlaufende Sequenz von Phasen, Aufgaben oder Ereignissen in einem zirkulären Fluss darzustellen. Betont die Verbindung zwischen allen Komponenten. Funktioniert am besten nur mit Level 1-Text. |

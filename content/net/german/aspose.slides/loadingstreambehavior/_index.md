## LoadingStreamBehavior Aufzählung

Der an eine Methode übergebene Stream wird als Binary Large Object (BLOB) betrachtet (siehe [`IBlobManagementOptions`](../iblobmanagementoptions) Beschreibung). Die Werte dieser Aufzählung geben an, wie der Stream behandelt werden soll, wenn er an die Methode übergeben wird. Je nach Anforderungen können unterschiedliche Entscheidungen getroffen werden, um das effizienteste Verhalten zu ermöglichen.

```csharp
public enum LoadingStreamBehavior
```

### Werte

| Name | Wert | Beschreibung |
| --- | --- | --- |
| ReadStreamAndRelease | `0` | Der Stream wird bis zum Ende gelesen und dann freigegeben – das heißt, es wird garantiert, dass dieser Stream in Zukunft nicht mehr von der [`IPresentation`](../ipresentation) Instanz verwendet wird. Er kann vom Client-Code geschlossen oder auf andere Weise verwendet werden. |
| KeepLocked | `1` | Der Stream wird im [`IPresentation`](../ipresentation) Objekt gesperrt, das heißt, das Eigentum am Stream wird übertragen. Das [`IPresentation`](../ipresentation) Objekt ist dafür verantwortlich, den Stream korrekt zu entsorgen, wenn dieses Objekt selbst entsorgt wird. Dieses Verhalten ist äußerst nützlich, wenn Sie eine große BLOB-Datei serialisieren müssen (wie ein großes Video- oder Audiodatei – siehe [`IBlobManagementOptions`](../iblobmanagementoptions) Beschreibung) und das Laden dieser Datei in den Speicher oder andere Leistungsprobleme vermeiden wollen. Sie können einfach den FileStream für diese Datei öffnen und an eine Methode übergeben, wobei Sie LoadingStreamBehavior KeepLocked wählen. |

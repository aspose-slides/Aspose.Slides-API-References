## PresentationLockingBehavior uppräkning

Representerar beteendet för hur [`IPresentation`](../ipresentation) källan (fil eller Stream) behandlas vid inläsning och arbete med en instans av [`IPresentation`](../ipresentation).

```csharp
public enum PresentationLockingBehavior
```

### Värden

| Namn | Värde | Beskrivning |
| --- | --- | --- |
| LoadAndRelease | `0` | Källan kommer endast att låsas under en tid av [`IPresentation`](../ipresentation) konstruktörutförande. Om [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed) är satt till false, kommer alla BLOBs att läsas in i minnet. Annars kan andra medel såsom temporära filer användas. Detta beteende är långsammare än KeepLocked, och om det är möjligt att överföra äganderätten för källan till [`IPresentation`](../ipresentation) rekommenderas att använda KeepLocked. |
| KeepLocked | `1` | Källan kommer att låsas under hela livslängden för en [`IPresentation`](../ipresentation) instans, tills den har frisläppts. [`IsTemporaryFilesAllowed`](../iblobmanagementoptions/istemporaryfilesallowed) måste vara satt till true för att använda detta beteende, annars kastas ett undantag. Detta beteende rekommenderas, det är snabbare och använder mindre minne än LoadAndRelease. |

### Anmärkningar

Källan är parametern som skickas till [`IPresentation`](../ipresentation)-konstruktören. I exemplet nedan är källan filen "pres.pptx":

```csharp
LoadOptions loadOptions = new LoadOptions { 
  BlobManagementOptions = { PresentationLockingBehavior = PresentationLockingBehavior.KeepLocked } };
using (IPresentation pres = new Presentation("pres.pptx", loadOptions)) { }
```

För detta exempel kommer källan ("pres.pptx"-filen) att låsas under en [`IPresentation`](../ipresentation)-instans livstid, dvs. den kan inte ändras eller tas bort av en annan process.

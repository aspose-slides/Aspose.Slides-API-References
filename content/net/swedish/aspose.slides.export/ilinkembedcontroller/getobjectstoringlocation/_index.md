## ILinkEmbedController.GetObjectStoringLocation metod

Bestämmer var objektet ska lagras. Denna metod anropas en gång för varje objekt-id. Det garanteras inte att det inte finns två objekt med samma data, semanticName och contentType men med olika id.

```csharp
public LinkEmbedDecision GetObjectStoringLocation(int id, byte[] entityData, string semanticName, 
    string contentType, string recomendedExtension)
```

| Parameter | Typ | Beskrivning |
| --- | --- | --- |
| id | Int32 | Objekt-id. Detta id är unikt för hela sparoperationen. |
| entityData | Byte[] | Objektets binära data. Denna parameter kan vara null, om objektets binära data ännu inte har genererats. |
| semanticName | String | Kort text som beskriver objektets betydelse. Kontrollern kan använda detta som en del av det externa objektets namn, men det är upp till dispatcher att säkerställa att namnen blir unika och endast innehåller tillåtna tecken. |
| contentType | String | MIME-typ för objektet. |
| recomendedExtension | String | Filnamnstillägg, rekommenderat för denna MIME-typ. |

## ILinkEmbedController.GetObjectStoringLocation método

Determina onde o objeto deve ser armazenado. Este método é chamado uma vez para cada id de objeto. Não há garantia de que não haverá dois objetos com os mesmos dados, semanticName e contentType, mas com ids diferentes.

```csharp
public LinkEmbedDecision GetObjectStoringLocation(int id, byte[] entityData, string semanticName, 
    string contentType, string recomendedExtension)
```

| Parâmetro | Tipo | Descrição |
| --- | --- | --- |
| id | Int32 | Id do objeto. Este id é único em toda a operação de gravação. |
| entityData | Byte[] | Dados binários do objeto. Este parâmetro pode ser nulo, se os dados binários do objeto ainda não foram gerados. |
| semanticName | String | Algum texto curto que descreve o significado do objeto. O controlador pode usar isso como parte do nome externo do objeto, mas cabe ao despachante garantir que os nomes sejam únicos e contenham apenas caracteres permitidos. |
| contentType | String | Tipo MIME do objeto. |
| recomendedExtension | String | Extensão de nome de arquivo recomendada para este tipo MIME. |

### Valor de Retorno

Decisão

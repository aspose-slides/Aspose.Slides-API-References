## LoadingStreamBehavior enumeração

O Stream passado para um método é considerado um Binary Large Object (BLOB) (veja a descrição [`IBlobManagementOptions`](../iblobmanagementoptions)). Os valores desta enumeração identificam como o Stream deve ser tratado quando passado para o método. Dependendo dos requisitos, diferentes decisões podem ser tomadas para fornecer o comportamento mais eficiente.

```csharp
public enum LoadingStreamBehavior
```

### Valores

| Nome | Valor | Descrição |
| --- | --- | --- |
| ReadStreamAndRelease | `0` | O stream será lido até o final e então liberado - ou seja, será garantido que este stream não será usado pela instância [`IPresentation`](../ipresentation) no futuro. Ele pode ser fechado pelo código cliente ou usado de qualquer outra forma. |
| KeepLocked | `1` | O stream será bloqueado dentro do objeto [`IPresentation`](../ipresentation), ou seja, a propriedade do stream será transferida. O objeto [`IPresentation`](../ipresentation) será responsável por descartar corretamente o stream quando este objeto for descartado. Este comportamento é extremamente útil quando você precisa serializar um grande arquivo BLOB (como um vídeo ou áudio grande - veja a descrição [`IBlobManagementOptions`](../iblobmanagementoptions)) e deseja impedir que esse arquivo seja carregado na memória ou causar outros problemas de desempenho. Você pode simplesmente abrir o FileStream para este arquivo e passá-lo para um método, escolhendo KeepLocked LoadingStreamBehavior. |

## LoadingStreamBehavior enumerazione

Lo Stream passato a un metodo è considerato come un Binary Large Object (BLOB) (vedi la descrizione [`IBlobManagementOptions`](../iblobmanagementoptions)). I valori di questa enumerazione identificano come lo Stream deve essere trattato quando viene passato al metodo. A seconda dei requisiti, possono essere prese decisioni diverse per fornire il comportamento più efficiente.

```csharp
public enum LoadingStreamBehavior
```

### Valori

| Nome | Valore | Descrizione |
| --- | --- | --- |
| ReadStreamAndRelease | `0` | Lo stream verrà letto fino alla fine e poi rilasciato - cioè sarà garantito che questo stream non sarà più utilizzato dall'istanza [`IPresentation`](../ipresentation) in futuro. Può essere chiuso dal codice client o utilizzato in qualsiasi altro modo. |
| KeepLocked | `1` | Lo stream verrà bloccato all'interno dell'oggetto [`IPresentation`](../ipresentation), cioè la proprietà dello stream sarà trasferita. L'oggetto [`IPresentation`](../ipresentation) sarà responsabile di liberare correttamente lo stream quando questo oggetto verrà eliminato. Questo comportamento è estremamente utile quando è necessario serializzare un grande file BLOB (ad esempio un video o audio di grandi dimensioni - vedi la descrizione [`IBlobManagementOptions`](../iblobmanagementoptions)) e si desidera evitare di caricare questo file in memoria o altri problemi di prestazioni. È sufficiente aprire il FileStream per questo file e passarlo a un metodo, scegliendo il LoadingStreamBehavior KeepLocked. |

## ILinkEmbedController.GetObjectStoringLocation metodo

Determina dove l'oggetto deve essere archiviato. Questo metodo viene chiamato una volta per ciascun id oggetto. Non è garantito che non esistano due oggetti con gli stessi dati, semanticName e contentType ma con id diversi.

```csharp
public LinkEmbedDecision GetObjectStoringLocation(int id, byte[] entityData, string semanticName, 
    string contentType, string recomendedExtension)
```

| Parametro | Tipo | Descrizione |
| --- | --- | --- |
| id | Int32 | Identificatore dell'oggetto. Questo identificatore è univoco a livello di operazione di salvataggio. |
| entityData | Byte[] | Dati binari dell'oggetto. Questo parametro può essere null, se i dati binari dell'oggetto non sono ancora stati generati. |
| semanticName | String | Un breve testo che descrive il significato dell'oggetto. Il controller può usarlo come parte del nome dell'oggetto esterno, ma spetta al dispatcher garantire che i nomi siano unici e contengano solo caratteri consentiti. |
| contentType | String | Tipo MIME dell'oggetto. |
| recomendedExtension | String | Estensione del nome file consigliata per questo tipo MIME. |

Decisione

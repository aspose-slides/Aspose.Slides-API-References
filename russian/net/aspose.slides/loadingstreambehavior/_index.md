## LoadingStreamBehavior enumeration

Stream переданный методу, рассматривается как большой двоичный объект (BLOB) (см. [`IBlobManagementOptions`](../iblobmanagementoptions) описание). Значения этого перечисления определяют, как Stream следует обрабатывать, когда он передается методу. В зависимости от требований могут быть приняты различные решения для обеспечения наиболее эффективного поведения.

```csharp
public enum LoadingStreamBehavior
```

### Ценности

| Имя | Ценность | Описание |
| --- | --- | --- |
| ReadStreamAndRelease | `0` | Поток будет прочитан до конца и затем освобожден - т.е. будет гарантировано, что этот поток не будет использован[`IPresentation`](../ipresentation) экземпляр в будущем. Его можно закрыть кодом клиента или использовать любым другим способом. |
| KeepLocked | `1` | Поток будет заблокирован внутри[`IPresentation`](../ipresentation) объект, т.е. право собственности на поток будет передано.[`IPresentation`](../ipresentation) объект будет отвечать за правильное удаление потока, когда этот объект будет удален сам. Это поведение чрезвычайно полезно, когда вам нужно сериализовать большой BLOB-файл (например, большой видео или аудио — см.[`IBlobManagementOptions`](../iblobmanagementoptions) описание) и хотите предотвратить загрузку этого файла в память или другие проблемы с производительностью. Вы можете просто открытьFileStream для этого файла и перейти к методу, выбравKeepLocked LoadingStreamBehavior. |
